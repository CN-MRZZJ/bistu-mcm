# 第一问：动检仪可靠性量化建模步骤

## 1. 建模目标
对每台动检仪（以`设备ID=车型_车号`区分）输出一个可解释的可靠性分数，并筛选异常设备。

- 输入：`csv_cleaned`中的清洗后数据
- 输出：
  - `reliability_score`（0-100，越高越可靠）
  - `is_abnormal_device`（是否异常设备）
  - 异常原因分解（漂移大、孤立大值多、冲突率高等）

## 2. 数据现状（基于当前附件）
- 总记录数：369,328
- 设备数：308
- 时间范围：2024-01-01 至 2024-04-09
- 三级及以上报警占比：0.233%
- 等级冲突占比：0.034%

说明：三级及以上样本少，直接做监督分类不稳。第一问建议以“无监督评分 + 规则筛选”为主。

## 3. 建模对象和粒度
以`设备ID`为分析单元，逐设备聚合特征。

- 设备ID定义：`车型 + "_" + 车号`
- 主键上下文：`线号 + 行别 + 里程 + 日期 + 设备ID`
- 空间邻域：同`线号+行别`下，里程±50m（0.05km）
- 时间邻域：3天滚动窗口（题干给出的判别周期）

## 4. 特征工程（核心）
建议构造四类特征，每类都要保留，保证可解释性。

### 4.1 一致性特征（最关键）
衡量设备与“其他设备对同一地点的反应”是否一致。

- `peer_residual_mean`：设备加速度与同地点同时间邻域“同类设备中位数”的平均偏差
- `peer_residual_p95`：上述偏差的95分位
- `peer_level_disagree_rate`：设备报警等级与群体主流等级不一致比例

意义：若某设备长期偏大或偏小，往往是设备状态问题而非线路问题。

### 4.2 异常触发行为特征
- `l3plus_rate`：`数据等级>=3`占比
- `isolated_l3plus_rate`：三级及以上中“3天内无其他设备复现”的比例
- `max_acc_spike_rate`：超过本车型速度修正后阈值的突刺比例

意义：异常设备常表现为“自己频繁报大值，但别人不报”。

### 4.3 稳定性与漂移特征
- `weekly_mean_slope`：按周聚合`max_acc`后做线性回归斜率
- `weekly_var_ratio`：后半段方差/前半段方差
- `day_night_gap`：昼夜工况下均值差异（控制车速后）

意义：器件老化/松动通常表现为慢性漂移或波动放大。

### 4.4 数据质量特征
- `conflict_rate`：`等级冲突标记`占比
- `valid_rows`：有效样本量
- `coverage_days`：覆盖天数

意义：样本量过低时不要贸然判异常，应降权或标注“证据不足”。

## 5. 评分体系（推荐）
先做分项分数，再融合成总分。

1. 对每个特征做稳健标准化：  
`z = (x - median) / MAD`
2. 将“越大越差”的特征映射为惩罚分`penalty_i`（如sigmoid截断到0-100）。
3. 总惩罚分：  
`total_penalty = Σ(w_i * penalty_i)`
4. 可靠性分：  
`reliability_score = 100 - total_penalty`

推荐初始权重（可调）：
- 一致性特征：0.40
- 异常触发行为：0.30
- 稳定性漂移：0.20
- 数据质量：0.10

## 6. 异常设备筛选规则
用“分数阈值 + 硬规则”双保险：

- 主规则：`reliability_score < P10`（设备分布第10百分位）判异常候选
- 硬规则A：`isolated_l3plus_rate`进入全体前5%
- 硬规则B：`peer_residual_p95`进入全体前5%
- 硬规则C：`conflict_rate`显著高于同线路设备（如>同线P95）

最终异常设备 = 主规则命中 或 任意2条硬规则命中。

## 7. 验证与稳健性检查
第一问通常无人工标签，建议做“弱验证”：

1. 时间切分验证：  
用1-2月建模、3-4月复核。看异常名单是否稳定。
2. 复现验证：  
异常设备在后续时间是否继续表现出高偏差和高孤立大值率。
3. 对照验证：  
同车型正常设备作为对照组，比较分布差异（KS检验/秩和检验）。
4. 灵敏度分析：  
调权重和阈值，观察异常名单变化幅度（Top-N稳定性）。

## 8. 实施顺序（落地）
1. 合并4个清洗后文件，统一按`日期`排序。  
2. 按`设备ID`构建上述特征表（每行一台设备）。  
3. 进行稳健标准化和总分计算。  
4. 生成异常设备清单与原因字段。  
5. 做时间切分复核，输出最终名单。

## 9. 交付物建议
- `q1_device_features.csv`：设备特征明细
- `q1_device_scores.csv`：设备分数与异常标记
- `q1_abnormal_devices.csv`：异常设备名单及触发原因
- `q1_model_note.md`：参数、阈值、验证结论

---
这套方案优点是：解释性强、和题干规则一致、可直接迁移到第二问实时算法中作为“设备可信度先验”。

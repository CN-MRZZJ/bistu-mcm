# 第一问论文稿：动检仪可靠性量化与异常设备筛选

## 摘要
针对题目“设计算法对每台动检仪可靠性进行量化并筛选异常设备”，本文提出一种“群体一致性 + 无监督聚类 + 规则复核”的方法。首先在同线同向、同时间、同里程邻域（50 米网格）上构造设备对照基准；其次提取系统偏差、非共识高报警、时间漂移、横纵向失衡、波动异常等特征；再采用按车型分层的稳健标准化与 DBSCAN 聚类得到设备可靠性分；最后引入“证据不足”机制降低小样本误判。实证结果显示，在 308 台设备中识别异常设备 35 台、证据不足设备 70 台、正常设备 203 台，可为后续实时告警判别提供设备先验可信度。

## 1. 问题重述
动检仪采集的振动加速度同时受线路状态、车型车况、车速、设备老化等因素影响。第一问目标是：
1. 给每台设备输出可比较的可靠性量化分值；
2. 从全体设备中筛出疑似异常设备。

难点在于：缺少人工标签，且高等级报警样本稀少，直接监督学习不稳定，因此采用无监督方法更可行。

## 2. 数据与预处理
数据来源于 `csv_cleaned`，包含 4 条线路、时间范围 2024-01-01 至 2024-04-09。  
预处理重点：
1. 过滤低速无效工况（车速 < 40 km/h）；
2. 去除重复记录；
3. 构造 `设备ID = 车型_车号`；
4. 生成 `max_acc = max(|垂加|, |水加|)`；
5. 里程按 50 米网格对齐（`50米网格里程`）。

## 3. 建模思路

### 3.1 同场景群体基准
以 `线号 + 行别 + 50米网格里程 + 日期` 作为同场景分组，计算：
- `peer_count`：同场景设备记录数；
- `peer_med_max_acc`：同场景 `max_acc` 中位数；
- `peer_med_level`：同场景 `数据等级` 中位数。

中位数而非均值的原因是抗异常值能力更强，能更稳定表示“多数设备共识”。

### 3.2 设备特征工程
对每台设备聚合得到 22 个特征，核心按 5 类组织：
1. 系统偏差：`peer_signed_residual_mean`、`peer_residual_mean`、`peer_residual_p95`；
2. 非共识高报警：`l3plus_rate`、`isolated_l3plus_rate`、`l3plus_cnt`；
3. 时间漂移：`weekly_mean_slope`、`piecewise_mean_gap`、`rolling_max_slope`；
4. 横纵向失衡：`vh_ratio_mean`、`vh_ratio_std`、`vh_level_gap_rate`；
5. 波动与质量：`max_acc_std`、`max_acc_p95`、`conflict_rate`、`peer_support_rate` 等。

### 3.3 分层标准化与聚类
由于车型动力学差异显著，先按主车型 (`dominant_vehicle_type`) 分层做 `RobustScaler`，小样本车型回退全局缩放。  
随后采用 DBSCAN 聚类：
- `min_samples = max(5, floor(sqrt(N_device)))`；
- 通过 k-distance 分位点网格搜索自动选取 `eps`；
- 噪声点（label = -1）作为重要异常信号。

### 3.4 可靠性分构建
将最大簇视为“主正常簇”，计算设备到该簇中心的欧氏距离，并线性映射为
`reliability_score ∈ [0,100]`（距离越远分越低）。

### 3.5 双门槛异常判定
设备满足以下任一条件即进入异常候选：
1. DBSCAN 噪声点；
2. `reliability_score <= P10`；
3. 规则命中数 `rule_hit_count >= 2`。  

规则集合（各取全体 P95 阈值）：
`peer_residual_p95`、`isolated_l3plus_rate`、`weekly_mean_slope`、`rolling_max_slope`、`vh_level_gap_rate`、`max_acc_std`。

### 3.6 证据不足机制
若设备满足 `n`、`coverage_days`、`peer_support_rate` 的低分位阈值（P10）条件，则标记为 `low_evidence`，不直接归入异常，降低小样本误报。

## 4. 结果
本次运行结果（`outputs/q1/q1_cluster_summary.txt`）：
- 总设备数：308
- 异常设备数：35
- 证据不足设备数：70
- 正常设备数：203
- DBSCAN 噪声设备：31
- 参数：`eps = 3.9218`，`min_samples = 17`
- 可靠性分阈值：`P10 = 35.48`

## 5. 方法优势与局限

### 5.1 优势
1. 无需标签，适合本题数据条件；
2. 结合业务规则（50 米邻域、群体复核）可解释性强；
3. 分层标准化可减弱车型差异干扰；
4. 引入 `low_evidence`，减少小样本误报。

### 5.2 局限
1. DBSCAN 对参数仍敏感，跨数据周期需复校；
2. 当前 `peer_support_rate` 低分位区分度有限；
3. 缺少现场检修真值标签，准确率只能做弱验证。

## 6. 结论
本文建立了可落地的第一问可靠性评估流程：以群体一致性构造设备偏差特征，采用分层稳健标准化与 DBSCAN 实现无监督筛查，并通过规则与证据机制控制误报。该方案已输出设备级可靠性分和异常清单，可直接作为第二问实时告警判断的设备可信度先验。
